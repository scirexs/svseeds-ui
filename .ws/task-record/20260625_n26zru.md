## Goal

Add a single, standalone `Image` component that lets a developer render "one
image" without hand-writing `<picture>` / `<source>` / `srcset` / `sizes`
plumbing. The caller pre-prepares their own alternate formats/sizes; the
component abstracts the element composition, bakes in accessible `alt`, lazy
loading, layout-shift mitigation, an optional placeholder, optional preload, and
load/error state reflected onto both `data-*` attributes and the SVS `variant`.

## Done when

- `<Image src alt />` renders an accessible, lazy-loaded image and type-checks.
- Multiple formats/art-direction via `sources` produce a `<picture>` with the
  right `<source>` elements and a fallback `<img>`.
- Load success sets `data-loaded` + `variant=ACTIVE`; load error sets
  `data-error` + `variant=INACTIVE`; neither set ŌćÆ caller's `variant`.
- `firstView` is a working shortcut over `loading` / `fetchpriority` / `preload`,
  with individual props still overriding it.
- `bunx vitest --run ./tests/Image.svelte.test.ts`, `bun run check`, and
  `bun run test` all pass.

## Background / constraints

- Headless library: the component owns **structure + behaviour + ARIA only**;
  appearance is the caller's CSS via `styling` (`SVSClass`) keyed by part/variant
  classes. Do not ship visual CSS.
- Follow the component shape in `coding-style.md` ┬¦14 and match `Button.svelte`'s
  idioms. `_core` needs no change ŌĆö say so in the implementation.
- This is an **independent** component: no context, no coordinator, no child.
  It does **not** appear in any `dep.json` composition (no intra-library import).

## Detailed changes

### (C-1) `<script module>` exports

**Where:** top of `Image.svelte`, mirroring `Button.svelte`'s `<script module>`.

**Required behavior:**
- The interface MUST be `export interface ImageProps extends Omit<HTMLImgAttributes, "src" | "alt" | "onload" | "onerror">` and declare these members (required props first per ┬¦12):
  - `src: string;` ŌĆö required.
  - `alt: string;` ŌĆö required; **empty string is allowed** for decorative images.
  - `sources?: ImageSource[];` ŌĆö optional `<source>` entries (see type below).
  - `placeholder?: string;` ŌĆö CSS background value (solid color or blur data-URI)
    shown until the image loads.
  - `firstView?: boolean;` ŌĆö above-the-fold shortcut (see C-3). Default falsy.
  - `preload?: boolean;` ŌĆö emit a preload link (see C-6).
  - `onload?: (ev: Event) => void;` and `onerror?: (ev: Event | string) => void;`
    ŌĆö re-dispatched after internal handling (type to match the `<img>` handler
    signatures from `svelte/elements`; pick the precise handler types, e.g.
    `EventHandler<Event, HTMLImageElement>`).
  - `attach?: Attachment<HTMLImageElement>;`
  - `element?: HTMLImageElement;` ŌĆö bindable.
  - `styling?: SVSClass;`
  - `variant?: SVSVariant;` ŌĆö bindable; default `VARIANT.NEUTRAL`.
- `width` / `height` / `loading` / `fetchpriority` / `decoding` / `srcset` /
  `sizes` / `crossorigin` / `referrerpolicy` etc. MUST NOT be declared as named
  props ŌĆö they remain in `HTMLImgAttributes` and flow through `{...rest}`.
- Declare the `ImageSource` type:
  ```ts
  export interface ImageSource {
    srcset: string;        // required
    type?: string;         // e.g. "image/avif"
    media?: string;        // art-direction media query
    sizes?: string;
    width?: number;
    height?: number;
  }
  ```
- Export the marker types/const:
  ```ts
  export type ImageReqdProps = "src" | "alt";
  export type ImageBindProps = "element" | "variant";
  export const _IMAGE_PRESET = "svs-image";
  ```
- Imports: value `{ VARIANT, PARTS, _fnClass } from "./_core"`; type imports
  `Snippet`? (not needed ŌĆö no snippet), `Attachment` from `svelte/attachments`,
  `HTMLImgAttributes` + the chosen handler types from `svelte/elements`,
  `SVSClass` / `SVSVariant` from `./_core`. Keep value/type import blocks
  separate and source-ordered (┬¦1).

### (C-2) Root anatomy ŌĆö always `<picture>`

**Where:** the template.

**Required behavior:**
- The root element MUST be `<picture class={[cls(PARTS.WHOLE, variant), c]}>`.
  Rendering `<picture>` even with zero `sources` is valid and keeps the part
  structure constant.
- Inside, for each `sources` entry render `<source>` with `srcset` (required) and
  `media` / `sizes` / `width` / `height` when present, **before** the `<img>`.
- **`type` default from extension (C-2a):** each `<source>`'s `type` MUST be the
  entry's explicit `type` when given, otherwise derived from the file extension of
  the **first candidate** in its `srcset`, using this exact map (case-insensitive
  extension; explicit `type` always wins):
  | ext | type |
  |---|---|
  | `avif` | `image/avif` |
  | `webp` | `image/webp` |
  | `svg` | `image/svg+xml` |
  | `jpg` / `jpeg` | `image/jpeg` |
  | `png` | `image/png` |
  | `apng` | `image/apng` |
  | `gif` | `image/gif` |
  When the extension is none of the above (or undeterminable), `type` MUST be
  omitted (no guess). Parse the first `srcset` candidate's URL (token before the
  first whitespace/comma), strip any `?`/`#` query/fragment, then read the
  extension. Implement this as a small local helper in the component (do **not**
  touch `_core.ts`); the same derivation is reused by the preload link (C-6).
- The `<img>` MUST carry `class={cls(PARTS.MAIN, variant)}`, `bind:this={element}`,
  `{src}`, `{alt}`, and `{@attach attach}`.
- `cls` MUST be `const cls = $derived(_fnClass(_IMAGE_PRESET, styling));`.

**Suggested implementation (Anatomy sketch):**
```svelte
<picture class="whole">
  <source srcset type media sizes>   <!-- conditional: per sources entry -->
  <img class="main" {src} {alt} loading fetchpriority decoding
       style onload onerror data-loaded data-error {...rest} {@attach attach}>
</picture>
```

### (C-3) Defaults & the `firstView` hybrid

**Where:** the `<img>` attributes and an `Initialize` block.

**Required behavior:**
- `firstView` is pure sugar; individual native attrs MUST still win:
  - The component MUST set, **before** `{...rest}` on the `<img>`:
    `loading={firstView ? "eager" : "lazy"}`,
    `fetchpriority={firstView ? "high" : undefined}`,
    `decoding="async"`.
  - Because these sit before `{...rest}`, a caller passing `loading` /
    `fetchpriority` / `decoding` (which land in `rest`) overrides the default.
    This ordering is the override mechanism ŌĆö do not reorder.
- `preload` is not an `<img>` attribute, so it is an explicit prop. The effective
  value MUST be `effPreload = preload ?? firstView ?? false` (explicit `preload`
  wins over `firstView`).

**Notes:** `undefined` `fetchpriority` must not emit the attribute (Svelte omits
`undefined` attrs).

### (C-4) Placeholder

**Where:** the `<img>` `style` and a `loaded` state.

**Required behavior:**
- While the image has not yet loaded **and** `placeholder` is set, the `<img>`
  MUST render `placeholder` as a CSS `background`. Once loaded it MUST be cleared.
- A caller-supplied `style` (via `rest`) MUST be preserved (merge, don't drop).

**Suggested implementation:** keep a `loaded` `$state(false)`; derive
`const bg = $derived(!loaded && placeholder ? \`background:${placeholder}\` : undefined);`.
To merge with a caller `style`, either declare `style` as an explicit prop and
compose `[bg, style].filter(Boolean).join(";") || undefined`, or compose against
`rest.style` ŌĆö choose the cleaner path; the required behavior above is the
contract.

**Risk:** background bleed if not cleared on load ŌĆö covered by a test.

### (C-5) Load / error state ŌåÆ `data-*` + `variant`

**Where:** `Event Handlers` + a small reactive/`Initialize` block.

**Required behavior:**
- On `load`: set internal `loaded` true and `errored` false; the `<img>` MUST
  expose `data-loaded` (and MUST NOT expose `data-error`); then re-dispatch the
  caller's `onload`.
- On `error`: set `errored` true and `loaded` false; the `<img>` MUST expose
  `data-error` (and MUST NOT expose `data-loaded`); then re-dispatch the caller's
  `onerror`.
- `variant` MUST be driven from state: **loaded ŌćÆ `VARIANT.ACTIVE`**,
  **error ŌćÆ `VARIANT.INACTIVE`**, **otherwise ŌćÆ the caller's `variant`**. Use the
  save/restore approach from `Button.svelte` (`prevVariant`): remember the
  caller's value before overriding, restore it when state returns to neutral.
- When `src` changes, internal state MUST reset (no `data-loaded`/`data-error`,
  caller `variant` restored) so the new image re-evaluates from scratch.
- Component-owned `data-loaded` / `data-error` MUST be placed **after**
  `{...rest}` so a caller cannot override them; the wrapped `onload` / `onerror`
  handlers likewise component-owned (the caller's are re-dispatched, not spread).

**Notes:** `data-*` and `variant` are intentionally redundant signals ŌĆö `data-*`
for direct CSS attribute targeting, `variant` for the SVS variant-class system.
Keep both.

**Risk:** the bindable `variant` save/restore can loop if written carelessly ŌĆö
follow Button's guarded `$effect` (only write when the value actually changes).

### (C-6) Preload link

**Where:** a `<svelte:head>` at the template top (per ┬¦14 skeleton position).

Follow the **responsive-image preload** approach from
`https://web.dev/articles/preload-responsive-images`: preload via the
`imagesrcset` / `imagesizes` link attributes (not a bare `href`) so the preloaded
resource matches the browser's responsive/`<picture>` selection, and preload the
**highest-priority candidate** ŌĆö i.e. the first `<source>` (`sources[0]`, the most
modern format the browser evaluates first) ŌĆö not the `<img>` fallback.

**Required behavior:**
- When `effPreload` is false, `<svelte:head>` MUST emit nothing.
- When `effPreload` is true, emit a single `<link rel="preload" as="image">`
  describing the candidate the browser would pick first:
  - **With `sources` (non-empty):** mirror `sources[0]` ŌĆö
    `imagesrcset={sources[0].srcset}`,
    `imagesizes={sources[0].sizes ?? <img sizes from rest>}`,
    `type={derived/explicit type of sources[0]}` (reuse the C-2a derivation),
    `media={sources[0].media}` when present.
  - **Without `sources`:** mirror the `<img>` ŌĆö `imagesrcset={rest.srcset}` and
    `imagesizes={rest.sizes}` when present.
  - In **both** cases include `href={src}` (the fallback for browsers without
    `imagesrcset` support) and `fetchpriority` when set (carry the effective
    fetch-priority ŌĆö `"high"` under `firstView`).
  - Omit any attribute whose value is absent (no empty `imagesizes` / `type` /
    `media`).

**Accepted limitation:** only the single highest-priority candidate (`sources[0]`
or the `<img>`) is preloaded; the component does not emit one preload `<link>` per
`<source>` for full art-direction (`media`/`type`) fan-out. State this in the
`@component` *Behavior*.

## Invariants / expected behavior

- Default render (no `firstView`) is `loading="lazy"`, `decoding="async"`, no
  `fetchpriority`, no preload link.
- `alt=""` is honored (decorative); `alt` is always present on the `<img>`.
- `variant` is bindable and reflects state per C-5; callers binding `variant`
  see ACTIVE on load and INACTIVE on error.

## Out of scope

- No wrapper `<div>`, no snippet-based placeholder, no `fill` mode (CSS-background
  placeholder only ŌĆö settled).
- No automatic image optimization / resizing / format conversion (caller prepares
  assets).
- No changes to `_core.ts`, `dep.json`, build scripts, or any other component.
- No dedicated `error` fallback snippet/src (error is surfaced via `data-error` +
  `variant=INACTIVE` only ŌĆö settled).

## Accepted risks / tradeoffs

- `data-*` and `variant` carry redundant state signals ŌĆö accepted for styling
  ergonomics (two styling surfaces).
- Preload does not model `<source>` selection (C-6 limitation) ŌĆö accepted.
- A caller passing `style` keeps full control of the img background; the
  placeholder relies on `style` merge, not an isolated layer ŌĆö accepted (no
  wrapper).
